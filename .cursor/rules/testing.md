# Стандарты тестирования

## Пирамида тестов

```
         /\
        /  \     E2E (мало)
       /────\    
      /      \   Интеграционные (часть)
     /────────\  
    /          \ Юнит-тесты (много)
   /────────────\
```

- **Юнит-тесты**: быстрые, изолированные, их много
- **Интеграционные**: проверяют связку модулей
- **E2E**: только критичные пользовательские сценарии

## Структура теста (AAA)

```typescript
it('should calculate total with discount', () => {
  // Arrange — подготовка данных
  const cart = new Cart();
  cart.addItem({ price: 100, quantity: 2 });
  
  // Act — выполнение кода
  const total = cart.calculateTotal(0.1);
  
  // Assert — проверка результата
  expect(total).toBe(180);
});
```

## Что тестировать

### Всегда

- Публичные API и функции
- Бизнес-логику
- Граничные случаи и обработку ошибок
- Преобразования данных

### Не тестировать

- Приватные методы (через публичный API)
- Код фреймворков и библиотек
- Интеграции со сторонними сервисами (мокать)
- Простые геттеры/сеттеры

## Моки

**Когда мокать:**
- внешние сервисы (API, БД)
- код, зависящий от времени
- случайные значения
- операции с файловой системой

**Когда не мокать:**
- тестируемый код
- простые чистые функции
- структуры данных
