# Правила безопасности (критично)

## НИКОГДА

- ❌ Коммитить в git секреты, API-ключи, токены
- ❌ Использовать `eval()` или `Function()` над пользовательским вводом
- ❌ Хранить пароли в открытом виде (используй bcrypt/argon2)
- ❌ Склеивать SQL строками (используй параметризованные запросы)
- ❌ Полагаться только на клиентскую валидацию

## ВСЕГДА

- ✅ Валидировать весь пользовательский ввод на сервере
- ✅ Использовать HTTPS в продакшене
- ✅ Добавлять `.env` в `.gitignore`

## Когда читать skill по безопасности

**Обязательно прочитай `.cursor/skills/security-guidelines/SKILL.md`, когда:**

- реализуешь аутентификацию/авторизацию
- создаёшь API-эндпоинты
- обрабатываешь пользовательский ввод или загрузку файлов
- работаешь с чувствительными данными (пароли, ПДн, платежи, токены)
- используешь внешние API
- реализуешь сессии или куки
