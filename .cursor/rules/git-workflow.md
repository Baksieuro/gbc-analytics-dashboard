# Рабочий процесс с Git

## Именование веток

```
<type>/<краткое-описание>
```

### Типы

| Тип | Назначение | Пример |
|------|------------|--------|
| `feature/` | Новая функциональность | `feature/user-auth` |
| `fix/` | Исправления багов | `fix/login-redirect` |
| `hotfix/` | Срочный фикс в проде | `hotfix/payment-crash` |
| `refactor/` | Улучшение кода | `refactor/api-layer` |
| `docs/` | Документация | `docs/api-readme` |
| `test/` | Добавление тестов | `test/auth-coverage` |
| `chore/` | Обслуживание | `chore/update-deps` |

### Правила

- только нижний регистр
- дефисы вместо пробелов
- коротко, но понятно
- при необходимости номер тикета: `feature/AUTH-123-oauth`

## Стратегия веток

- `main` — код, готовый к продакшену
- `develop` — интеграционная ветка (опционально)
- Фичевые ветки сливаются в main через PR

## Практика коммитов

### Атомарные коммиты

- один логический смысл на коммит
- проект собирается на каждом коммите
- тесты проходят на каждом коммите

### Перед коммитом

1. Запустить тесты
2. Запустить линтер
3. Просмотреть изменения: `git diff`

## Рекомендации по pull request

### Перед созданием PR

- [ ] Тесты проходят
- [ ] Код просмотрен вами
- [ ] Ветка актуальна относительно main
- [ ] Нет конфликтов слияния

### Размер PR

- **Желательно**: меньше 400 строк
- **Максимум**: 1000 строк
- Крупные изменения разбивать на несколько PR

## Не делай

- ❌ Force push в `main`
- ❌ Коммить секреты и учётные данные
- ❌ Коммить крупные бинарные файлы
- ❌ Создавать огромные PR
- ❌ Оставлять WIP-коммиты в PR
- ❌ Сливать без ревью

## Делай

- ✅ Пиши осмысленные сообщения коммитов
- ✅ Держи PR сфокусированными
- ✅ Обновляй ветку перед слиянием
- ✅ Удаляй ветку после слияния

## Когда использовать skill git-helper

**Используй `.cursor/skills/git-helper/SKILL.md`, когда:**

- нужно сгенерировать сообщение коммита по индексированным изменениям
- разрешаешь конфликты слияния
- нужны конкретные команды git (cherry-pick, stash, bisect, rebase)
- создаёшь или ведёшь ветки
- отменяешь операции (reset, revert, restore)
- разбираешься с историей (blame, log, bisect)
